Chrome Build Instructions

The correct Chromium revision to build against is revision 50686 (which matches the point where the WebKit audio branch was taken). For those who want more information, normal instructions for building Chrome are here:


  1. Create a directory called 'Chromium' (or whatever you want to call it)

  2. cd into the directory and get the Chromium source from the specific revision 50686 
    3. $ cd Chromium
$ gclient config http://src.chromium.org/svn/trunk/src
$ svn co -r 50686 http://src.chromium.org/svn/trunk/src

  3. You then need to change your .gclient file to the following: 
    4. # An element of this array (a "solution") describes a repository directory
# that will be checked out into your working copy.  Each solution may
# optionally define additional dependencies (via its DEPS file) to be
# checked out alongside the solution's directory.  A solution may also
# specify custom dependencies (via the "custom_deps" property) that
# override or augment the dependencies specified by the DEPS file.
solutions = [
  { "name"        : "src",
  # "url"         : "http://src.chromium.org/svn/trunk/src",
   "url"         : "http://src.chromium.org/svn/trunk/src@50686",

    # "url"         : None,
    "custom_deps" : {
      # To use the trunk of a component instead of what's in DEPS:
      #"component": "https://svnserver/component/trunk/",
      # To exclude a component from your working copy:
      #"data/really_large_component": None,
      "src/third_party/WebKit": None,
      "src/third_party/WebKit/JavaScriptCore": None,
      "src/third_party/WebKit/WebCore": None,
      "src/third_party/WebKit/WebKitLibraries": None,
      "src/third_party/WebKit/WebKit/mac": None,
      "src/third_party/WebKit/WebKitTools": None,
      "src/third_party/WebKit/WebKitTools/Scripts": None,
      "src/third_party/WebKit/LayoutTests": None,
      "src/third_party/WebKit/WebKit/chromium": None,
      "src/webkit/data/layout_tests/LayoutTests": None,
    }
  }
]

  4. cd into the 'src' directory

  5. Edit the DEPS file so that gclient will NOT automatically fetch the WebKit directory

  6. Run 'gclient sync' which pulls down a bunch of other code from other repositories that Chrome uses 
    7. $ gclient sync

  7. Go into the 'third_party" directory and use subversion directly (not gclient) to get the WebKit source from the special audio branch of WebKit 
    8. $ cd third_party
$ svn co https://svn.webkit.org/repository/webkit/branches/audio WebKit

  8. You may not need to do this, but I found that I had to edit this file: 
    9. Chromium/src/chrome/browser/webdata/web_database.cc
and return immediately from the WebDatabase::MigrateOldVersionsAsNeeded() method (don't do anything in the function)

  9. At this point you may need to re-run gclient to generate your Xcode projects:
    10. $ gclient runhooks --force

  10. You can now open the chrome.xcodeproj project (in the src/chrome directory) and build.
